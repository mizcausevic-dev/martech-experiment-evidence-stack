# Security

## Data handling

- This repo uses synthetic sample data only.
- Do not commit live tenant exports, audience lists, subscriber records, consent logs, warehouse extracts, API keys, or webhook secrets.
- Keep any future connector work read-only and least-privilege.

## Product claims

- The repo demonstrates Growth Ops evidence-routing and activation-governance patterns, not a production tenant control plane.
- Do not claim compliance, certification, or hosted readiness unless those states are actually true.

## Disclosure

If you find a security issue in the repo itself, open a private report through GitHub security advisories or contact the maintainer directly before public disclosure.
